## What is Tig?

Tig is a platform that automatically captures AI conversations from tools like **Cursor**, **Claude Code**, and **Windsurf**, linking them to your commits and pull requests. It provides your team with searchable context for every line of code built with AI assistance.

## How It Works

<Steps>
<Step title="Install Tig">
  Install the NPM package and authenticate with GitHub

  ```bash
  npm install -g @tigtech/tig-cursor
  tig login
  ```
</Step>

<Step title="Set Up Your Project">
  Run setup in your project directory to configure automatic capturing

  ```bash
  cd /path/to/your/project
  tig setup
  ```
</Step>

<Step title="Install GitHub App">
  Install the Tig Connector GitHub App to enable PR view automation and state tracking
</Step>

<Step title="Code as Usual">
  Continue using your AI assistant—Tig automatically captures conversations on commit and push
</Step>

<Step title="Review PRs with Context">
  When you open a PR, Tig posts a comment with a link to view all AI interactions chronologically
</Step>
</Steps>

## Key Features

### Tig Blame
Click any line of code in your Tig dashboard to see which AI conversations led to its creation—similar to `git blame` but for AI context.

[Learn more about Tig Blame →](/essentials/tig-blame)

### MCP Tool
Fetch context from previous conversations directly in your AI assistant using the Model Context Protocol.

[Set up MCP →](/quickstart-mcp)

### State Tracking
The Tig Connector keeps your PR metadata in sync with GitHub, automatically updating states (open/closed/merged) and timestamps.

[Install Tig Connector →](/quickstart-connector)

### Code Review
When you open a pull request, the Tig Connector automatically posts a comment with a link to view all AI interactions from commits in the PR. Perfect for code reviews and understanding development context.

[Learn more about PR View →](/essentials/pr-view)

## Who Should Use Tig?

### Development Teams
- **Code reviewers** - Understand why code was written a certain way
- **New team members** - Learn from past AI conversations
- **Tech leads** - Track how AI is being used across the team

### Solo Developers
- **Context continuity** - Remember what you built and why
- **Knowledge base** - Search your own AI conversation history
- **Documentation** - Auto-document complex decisions made with AI
